Abstract

MathML is a W3C recommendation that represents mathematical expressions. This note describes a way to add semantic information to presentation markup.

Definitions

The intent of an element is a functional expression that captures the meaning conveyed by the author. It should also reflect the meaning that would be understood by a human reader. The intent may often be inferred from the presentation, especially when the author supplies information such as the subject area.

The intent attribute encodes the intent of an element in terms of the intent of its sub elements. It may contain an operator name, which may be followed by a parenthesized list of zero or more arguments. An operator name creates a function application when it is followed by a list of arguments, or when the element contains one or more sub elements with intent. Otherwise, the name refers to a mathematical constant or function in isolation.

The arg attribute provides a numeric index or a symbolic name for a sub element that may be used within an enclosing intent attribute to refer to the intent of the sub element. The form of an argument reference is the "@" sign, followed by the numeric index or symbolic name.

The default intent of an atomic element mi or mn is taken from the character data of the element. Other rules for inferring default intent for known notational patterns may be defined by the consumer of the markup. These rules may be influenced by subject area information provided by the author.

Examples -- Operator Name

The operator name and the existence of arguments determine the intent. The intent attribute may express:

  • an operator name, with arguments collected from the sub elements
  • a mathematical constant with no arguments
  • a mathematical function in isolation
  • a mathematical function with zero arguments

Function application

The intent attribute may express an operator name, with arguments collected from the sub elements.

$a+b$ as $plus(a,b)$

<mrow intent="plus">
  <mi>a</mi>
  <mo>+</mo>
  <mi>b</mi>
</mrow>
Mathematical constant

The intent attribute may express a mathematical constant with no arguments.

$e$ as $exponentiale$

<mi intent="exponentiale">e</mi>
Function in isolation

The intent attribute may express a function in isolation, where the arguments have been omitted or supplied elsewhere.

$T$ as $transpose$

<mo intent="transpose">T</mo>
Function with zero arguments

The intent attribute may express a function with zero arguments.

$I$ as $identity()$

<mi intent="identity()">I</mi>
Examples -- Argument List

The argument list determines the sub structures of the intent. It may be omitted, or may contain zero, one, or more intent values:

  • missing arguments express a constant or function name
  • zero arguments express a function of zero arguments
  • one argument expresses a function of one argument
  • multiple comma-separated arguments express a function of multiple arguments

Missing argument list

An name with no argument list, and no sub elements with intent, expresses a constant or function name.

$e$ as $exponentiale$

<mi intent="exponentiale">e</mi>
Empty argument list

An empty argument list expresses a function with zero arguments.

$I$ as $identity()$

<mi intent="identity()">I</mi>
One argument list

One intent expression within open and close parentheses expresses a function with one argument.

$n!$ as $factorial(n)$

<mrow intent="factorial(@1)">
  <mi>n</mi>
  <mo>!</mo>
</mrow>
Multiple arguments

Multiple comma-separated intent expressions describe a function with multiple arguments.

$a+b$ as $plus(a,b)$

<mrow intent="plus(@1,@2)">
  <mi>a</mi>
  <mo>+</mo>
  <mi>b</mi>
</mrow>
Examples -- Argument Index

The direct sub elements with intent form the actual argument list. Each actual argument may be referenced by its position in the actual argument list:

  • index one refers to the intent of the first actual argument
  • index two refers to the intent of the next actual argument
  • index zero refers to the intent of the actual operator
  • the argument index may reorder the actual arguments

Index one

Index one refers to the intent of the first actual argument.

$n!$ as $factorial(n)$

<mrow intent="factorial(@1)">
  <mi>n</mi>
  <mo>!</mo>
</mrow>
Index two

Index two refers to the intent of the next actual argument.

$a+b$ as $plus(a,b)$

<mrow intent="plus(@1,@2)">
  <mi>a</mi>
  <mo>+</mo>
  <mi>b</mi>
</mrow>
Index zero

Index zero refers to the intent of the actual operator.

$n!$ as $factorial(n)$

<mrow intent="@0(@1)">
  <mi>n</mi>
  <mo intent="factorial">!</mo>
</mrow>
Argument order

The argument index references may reorder the actual arguments.

$\int_0^1 x^2 dx$ as $integral(power(x,2),x,0,1)$

<mrow intent="@0(@3,@4,@1,@2)">
  <msubsup>
    <mo intent="integral">&x222B;</mo>
    <mn>0</mn>
    <mn>1</mn>
  </msubsup>
  <msup intent="power">
    <mi>x</mi>
    <mn>2</mn>
  </msup>
  <mtext>d</mtext>
  <mi>x</mi>
</mrow>
Examples -- Argument Name

A sub element may be assigned an argument name that may be used as an alternative way to reference its intent within its outer element:

  • the arg attribute defines an argument name within an expression
  • the argument name may be used within the outer intent attribute
  • the operator may be assigned an argument name
  • the argument name may reorder the actual arguments

Argument name

The arg attribute defines an argument name within an expression.

$n!$ as $factorial(n)$

<mrow intent="factorial(@x)">
  <mi arg="x">n</mi>
  <mo>!</mo>
</mrow>
Argument reference

The argument name may be used within the outer intent attribute.

$a+b$ as $plus(a,b)$

<mrow intent="plus(@x,@y)">
  <mi arg="x">a</mi>
  <mo>+</mo>
  <mi arg="y">b</mi>
</mrow>
Operator name

The operator may be assigned an argument name.

$n!$ as $factorial(n)$

<mrow intent="@fn(@x)">
  <mi arg="x">n</mi>
  <mo arg="fn" intent="factorial">!</mo>
</mrow>
Argument order

The argument name references may reorder the actual arguments.

$\int_0^1 x^2 dx$ as $integral(power(x,2),x,0,1)$

<mrow intent="@op(@fn,@var,@lo,@hi)">
  <msubsup>
    <mo arg="op" intent="integral">&x222B;</mo>
    <mn arg="lo">0</mn>
    <mn arg="hi">1</mn>
  </msubsup>
  <msup arg="fn" intent="power(@base,@exp)">
    <mi arg="base">x</mi>
    <mn arg="exp">2</mn>
  </msup>
  <mtext>d</mtext>
  <mi arg="var">x</mi>
</mrow>
Examples -- Superscript

When the same notation may express different intent in printed material, the author and reader typically use other means to agree on the intent, such as notational conventions or expository text. For online markup, the intent can resolve the ambiguity directly. For example, the superscript notation may intend:

  • the most common use to denote exponentiation
  • another important use to index into tensors or matrices
  • one of several operators found in the superscript itself

Superscript as exponent

The most common use of superscript is to denote exponentiation.

$x^2$ as $power(x,2)$

<msup intent="power">
  <mi>x</mi>
  <mn>2</mn>
</msup>

$x^n$ as $power(x,n)$

<msup intent="power">
  <mi>x</mi>
  <mi>n</mi>
</msup>
Superscript as tensor or matrix index

Another important use to index into tensors or matrices.

$c_ix^i$ as $times(index(c,i),index(x,i))$

<mrow intent="times">
  <msub intent="index">
    <mi>c</mi>
    <mi>i</mi>
  </msub>
  <mo>&#x2062;</mo>
  <msup intent="index">
    <mi>x</mi>
    <mi>i</mi>
  </msup>
</mrow>
Superscript as the operator

The superscript itself may supply the operator.

$z^*$ as $conjugate(z)$

<msup intent="conjugate">
  <mi>z</mi>
  <mo>*</mo>
</msup>

<msup intent="@0">
  <mi>z</mi>
  <mo intent="conjugate">*</mo>
</msup>

$A^T$ as $transpose(A)$

<msup intent="transpose">
  <mi>A</mi>
  <mo>T</mo>
</msup>

<msup intent="@0">
  <mi>A</mi>
  <mo intent="transpose">T</mo>
</msup>

$A^\dagger$ as $adjoint(A)$

<msup intent="adjoint">
  <mi>A</mi>
  <mo>†</mo>
</msup>

<msup intent="@0">
  <mi>A</mi>
  <mo intent="adjoint">†</mo>
</msup>
Examples -- Binomial

The same intent may be expressed by different printed notations. For example, the binomial coefficient may be expressed as:

  • a stack of two numbers, enclosed in parentheses
  • a literal C with subscript and superscript

Binomial as stacked numbers

One notation for the binomial coefficient is as a stack of two numbers, enclosed in parentheses.

$\binom{n}{m}$ as $binom(n,m)$

<mfenced intent="binom">
  <msubsup>
    <mrow/>
    <mi arg="2">m</mi>
    <mi arg="1">n</mi>
  </msubsup>
</mfenced>
Binomial as subscript and superscript

Another notation for the binomial coefficient is as a literal C with subscript and superscript.

$C^n_m$ as $binom(n,m)$

<msubsup intent="binom">
  <mo>C</mo>
  <mi arg="2">m</mi>
  <mi arg="1">n</mi>
</msubsup>

<msubsup intent="@0">
  <mo intent="binom">C</mo>
  <mi arg="2">m</mi>
  <mi arg="1">n</mi>
</msubsup>
Examples -- Multiple Operators

When the presentation supplies an element for each intended structure, the intent attribute can be placed naturally on the element that presents the structure. When the presentation combines multiple structures into a single element, that element can be marked with the intent of the multiple structures:

  • multiple infix operators
  • unary prefix operators
  • postfix operators

One mrow with multiple infix operators

A single mrow that contains multiple infix operators can be marked with the intent of the multiple infix operators.

$a+b-c+d$ as $plus(a,plus(minus(b,c),d))$

<mrow intent="@p1(@a,@p2(@m(@b,@c),@d))">
  <mi arg="a">a</mi>
  <mo arg="p1" intent="plus">+</mo>
  <mi arg="b">b</mi>
  <mo arg="m" intent="minus">-</mo>
  <mi arg="c">c</mi>
  <mo arg="p2" intent="plus">+</mo>
  <mi arg="d">d</mi>
</mrow>

$a+b-c+d$ as $plus(plus(a,minus(b,c)),d)$

<mrow intent="@p2(@p1(@a,@m(@b,@c)),@d)">
  <mi arg="a">a</mi>
  <mo arg="p1" intent="plus">+</mo>
  <mi arg="b">b</mi>
  <mo arg="m" intent="minus">-</mo>
  <mi arg="c">c</mi>
  <mo arg="p2" intent="plus">+</mo>
  <mi arg="d">d</mi>
</mrow>

The nesting can also be made clear with additional mrow elements:

$a+b-c+d$ as $plus(a,plus(minus(b,c),d))$

<mrow intent="plus">
  <mi>a</mi>
  <mo>+</mo>
  <mrow intent="plus">
    <mrow intent="minus">
      <mi>b</mi>
      <mo>-</mo>
      <mi>c</mi>
    </mrow>
    <mo>+</mo>
    <mi>d</mi>
  </mrow>
</mrow>

$a+b-c+d$ as $plus(plus(a,minus(b,c)),d)$

<mrow intent="plus">
  <mrow intent="plus">
    <mi>a</mi>
    <mo>+</mo>
    <mrow intent="minus">
      <mi>b</mi>
      <mo>-</mo>
      <mi>c</mi>
    </mrow>
  </mrow>
  <mo>+</mo>
  <mi>d</mi>
</mrow>
One mrow with unary prefix operators

A single mrow that contains unary prefix operators can be marked with the intent of the unary prefix operators.

$a+b-c+d$ as $plus(a,b,minus(c),d)$

<mrow intent="@p1(@a,@b,@m(@c),@d)">
  <mi arg="a">a</mi>
  <mo arg="p1" intent="plus">+</mo>
  <mi arg="b">b</mi>
  <mo arg="m" intent="minus">-</mo>
  <mi arg="c">c</mi>
  <mo arg="p2" intent="plus">+</mo>
  <mi arg="d">d</mi>
</mrow>

<mrow intent="plus">
  <mi>a</mi>
  <mo>+</mo>
  <mi>b</mi>
  <mrow intent="minus">
    <mo>-</mo>
    <mi>c</mi>
  </mrow>
  <mo>+</mo>
  <mi>d</mi>
</mrow>
One mrow with unary postfix operators

A single mrow that contains unary postfix operators can be marked with the intent of the unary postfix operators.

$a+b!$ as $plus(a,factorial(b))$

<mrow intent="@p(@a,@f(@b))">
  <mi arg="a">a</mi>
  <mo arg="p" intent="plus">+</mo>
  <mi arg="b">b</mi>
  <mo arg="f" intent="factorial">!</mo>
</mrow>

<mrow intent="plus">
  <mi>a</mi>
  <mo>+</mo>
  <mrow intent="factorial">
    <mi>b</mi>
    <mo>!</mo>
  </mrow>
</mrow>
Examples -- Integrals of Fractions

Integrals of fractions are often presented with the differential $dx$ as part of the fraction. For these cases, the intent of the integral uses a named argument reference to find the bound variable, and special care to arrange for the correct intent for the integrand:

  • Differential alone in the numerator
  • Differential in a binary product
  • Differential in an nary product
  • Differential next to an exponent

Differential alone in the numerator

The intent of the fraction uses the numeral 1 as the numerator, and no other elements contribute to the numerator.

$\int_0^1 {dx \over x^2+1}$ as $integral(divide(1,plus(power(x,2),1)),x,0,1)$

<mrow intent="@0(@3,@bvar,@1,@2)">
  <msubsup>
    <mo intent="integral">&x222B;</mo>
    <mn>0</mn>
    <mn>1</mn>
  </msubsup>
  <mfrac intent="divide">
    <mrow intent="1">
      <mtext>d</mtext>
      <mi intent="bvar">x</mi>
    </mrow>
    <mrow intent="plus">
      <msup intent="power">
        <mi>x</mi>
        <mn>2</mn>
      </msup>
      <mo>+</mo>
      <mn>1</mn>
    </mrow>
  </mfrac>
</mrow>
Differential in a binary product

The intent of the fraction uses the initial $x$ as the numerator, and no other elements contribute to the numerator.

$\int_0^1 {x\,dx \over x^2+1}$ as $integral(divide(x,plus(power(x,2),1)),x,0,1)$

<mrow intent="@0(@3,@bvar,@1,@2)">
  <msubsup>
    <mo intent="integral">&x222B;</mo>
    <mn>0</mn>
    <mn>1</mn>
  </msubsup>
  <mfrac intent="divide">
    <mrow>
      <mi>x</mi>
      <mtext>d</mtext>
      <mi intent="bvar">x</mi>
    </mrow>
    <mrow intent="plus">
      <msup intent="power">
        <mi>x</mi>
        <mn>2</mn>
      </msup>
      <mo>+</mo>
      <mn>1</mn>
    </mrow>
  </mfrac>
</mrow>
Differential in an nary product

The intent of the fraction uses $2x$ as the numerator, and no other elements contribute to the product.

$\int_0^1 {2x\,dx \over x^2+1}$ as $integral(divide(times(2,x),plus(power(x,2),1)),x,0,1)$

<mrow intent="@0(@3,@bvar,@1,@2)">
  <msubsup>
    <mo intent="integral">&x222B;</mo>
    <mn>0</mn>
    <mn>1</mn>
  </msubsup>
  <mfrac intent="divide">
    <mrow intent="times">
      <mn>2</mn>
      <mi>x</mi>
      <mtext>d</mtext>
      <mi intent="bvar">x</mi>
    </mrow>
    <mrow intent="plus">
      <msup intent="power">
        <mi>x</mi>
        <mn>2</mn>
      </msup>
      <mo>+</mo>
      <mn>1</mn>
    </mrow>
  </mfrac>
</mrow>
Differential next to an exponent

The intent of the fraction uses $x^2$ as the numerator, and no other elements contribute to the numerator.

$\int_0^1 {x^2\,dx \over x^2+1}$ as $integral(divide(power(x,2),plus(power(x,2),1)),x,0,1)$

<mrow intent="@0(@3,@bvar,@1,@2)">
  <msubsup>
    <mo intent="integral">&x222B;</mo>
    <mn>0</mn>
    <mn>1</mn>
  </msubsup>
  <mfrac intent="divide">
    <mrow>
      <msup intent="power">
        <mi>x</mi>
        <mn>2</mn>
      </msup>
      <mtext>d</mtext>
      <mi intent="bvar">x</mi>
    </mrow>
    <mrow intent="plus">
      <msup intent="power">
        <mi>x</mi>
        <mn>2</mn>
      </msup>
      <mo>+</mo>
      <mn>1</mn>
    </mrow>
  </mfrac>
</mrow>
